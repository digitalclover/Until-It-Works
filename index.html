Tabbed Content

May 25, 2016

As my first entry on this web development blog, I decided to start off on an easy topic, discussing something that I have seen countless of times when working with other companies and developers. Tabbed content is an easy design solution when you have a grouping of content based on category. Visually, it makes sense to the user that what tab they tap on will display the content of that category.

Now this is by no means a bash on Bootstrap. It is an impressive resource that has its uses for very specific circumstances, however I do not find production websites to be one of them.

If you were to look at the supplemented source code that accompanies this tab example, you will find something that I see from many developers. Note that I've cleaned out the majority of classes to make things more ledgible.

This is the most basic example of a developer trying to make the code more convenient for them, while sacrificing the convenience of the user.

The easiest example of this inconvenience is screen readers. A screen reader, for those unaware, is a browser tool that audibly reads the content from a webpage. The most common user case for screen readers are for those with impared vision.

Screen readers ignore any CSS since there is no use for visual styles. The same argument can be made for JavaScript functions that visually change the same content. Take away the CSS and JavaScript from the above example, and you are left with the following:

Home, Profile, Messages, Settings,
Content, Content, Content, Content

Now replace each Content with paragraphs upon paragraphs of text. Will you know when one section ends? How about when another section begins? This current setup presents no guidance or organization of content.

So how do we fix this? I’ll fill you in on some options tomorrow.
